# Object-Oriented Programming (OOPS)

This package covers the four pillars of OOPS and demonstrates object-oriented design principles.

## Topics Covered

- **Abstraction** — Hiding implementation details and showing only necessary features
- **Encapsulation** — Bundling data and methods, using access modifiers (private, public, protected)
- **Inheritance** — Extending classes and reusing code through parent-child relationships
- **Polymorphism** — Objects taking multiple forms, method overriding, and method overloading

## Key Principles

1. **Abstraction** 
   - Hide complexity from users
   - Show only relevant features
   - Use abstract classes and interfaces

2. **Encapsulation**
   - Keep data private
   - Provide public getter/setter methods
   - Control data access and modification

3. **Inheritance**
   - Parent class provides common functionality
   - Child class extends and specializes
   - Promotes code reuse and hierarchy

4. **Polymorphism**
   - Same method name, different implementations
   - Compile-time: Method overloading
   - Runtime: Method overriding
